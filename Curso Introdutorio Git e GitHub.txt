# Anotações sobre Git/GitHub## Git- Na sua descrição mais simples, significa Software de Controle de Versão (VCS) > distribuido. Feito basicamente para Versionamento de código. Pode ser usado para outras coisas também, mas em sua maioria é isso.- Repositório local que permite que os devs criem seus codigos que futuramente serão enviados para o Repositório remoto (GitHub)- Controle de histórico de todos os comitts- Trabalho em equipe: o versionamento permite que mais de uma pessoa acesse e faça contribuições ao seu código- Ramificação de projeto: melhor distriuição de funções para determinados devs por projetos, depois pode juntar as ramificações e finalizar o codigo.- Segurança: funções bem delimitadas que permete um certo sigilo a sua área de atuação - Organização, limpeza e praticidade: ecomonia de trabalho para evitar criação de inumeros zip's- A cada commit, cada mudança no seu dado, gera uma nova chave SSH - Difere de GitHub ## GitHub- Repositório remoto mais usado no mundo- Local para guardar os arquivos de forma social para programadores- Comunidade com caracteristicas de rede social- Repositorio publico para demonstração profissional, muito usado pelas empresas para nivelar o candidato para determinada vaga, pois lá você pode acompanhar os projetos, os repositórios do usuário- Open source (software de código aberto/livre)- Repositorios ilimitados (para repositorio privado permite que você trabalhe com até 3 pessoas)- Hospedagem de cod-fonte - Forks > colaboração de projetos alheios por conta propria (segundo algumas regras eticas)## Complentacões Gerais - GitHub Desktop: versão que simplifica commits, push e etc. Plataforma da propria Microsoft {https://desktop.github.com/}- commit > quando você torna efetiva uma alteração feita no seu código- Link uteis [Comandos basicos git]{https://gist.github.com/leocomelli/2545add34e4fec21ec16}
